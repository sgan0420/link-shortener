## 1. Short URL path: the chosen solution

Every short URL is a **7-character slug generated by [nanoid](https://github.com/radeno/nanoid.rb)** over its default URL-safe alphabet (`A‚ÄďZ`, `a‚Äďz`, `0‚Äď9`, `_`, `-`). At length 7 the keyspace is 64‚Ā∑ ‚Čą **4.4 √ó 10¬Ļ¬≤** distinct values. Slugs are persisted with a unique index on `short_links.slug`; collisions resolve via application-level retry (3 attempts) before raising `UrlShortener::CollisionExhaustedError`.

### Why nanoid over `Base62(id)`?

- **Unguessability.** Sequential ID-based slugs leak total link count and let attackers walk the namespace ‚ÄĒ privacy and abuse risk. Nanoid slugs are uniformly random.
- **No coupling to the primary key.** A future re-platforming (cross-shard IDs, snowflake IDs) doesn't change the public URL surface.

### Generation flow

```
UrlShortener.call(target_url:)
  validate scheme (http/https) + length (‚Č§2048) + URI.parse-ability
  3.times:
    slug = Nanoid.generate(size: 7)
    insert short_links(slug:, target_url:, title_status: :pending)
    return on success
    next on slug-taken (race against unique index)
    raise ValidationError on any other validation failure
  raise CollisionExhaustedError
```

The DB unique index is the source of truth; the app-level retry exists to keep UX clean during the narrow race window between two concurrent inserts picking the same nanoid.

## 2. Limitations and workarounds

### 2.1 Collision probability (birthday paradox)

For a keyspace of size N = 64‚Ā∑, the probability of any collision among k slugs is roughly k¬≤ / (2N). The 50 % collision threshold sits near ‚ąöN ‚Čą 1.9 M slugs ‚ÄĒ well past assessment-relevant scale, but a real concern for a long-running production system.

**Workaround at scale:** extend the slug length by one character (64‚Āł ‚Čą 2.8 √ó 10¬Ļ‚Āī). Already a one-constant change in `UrlShortener::SLUG_LENGTH`.

### 2.2 Reserved slugs

`up`, `stats`, `about`, `admin`, `rails`, `assets`, `short_links`, `my-links` are blocked at creation time by `ShortLink::RESERVED_SLUGS`. The DB unique index can't protect against future top-level routes shadowing user-generated slugs ‚ÄĒ this list is the application-level invariant. New top-level routes must add themselves here.

### 2.3 Public stats pages

Intentional for this demo: anyone with a short URL can append `/stats` and see traffic. A production deployment would gate this behind ownership/auth (magic-link sign-in, OAuth, etc. ‚ÄĒ see [`docs/BACKLOG.md`](./BACKLOG.md)).

### 2.4 Geolocation accuracy

The geolocation provider is **ipapi.co** ‚ÄĒ free, no signup, HTTPS on the free tier. Real-world inaccuracies:

- VPN / proxy traffic resolves to the proxy's location, not the user's.
- Mobile carrier NAT sometimes returns the carrier hub city.
- Free tier rate-limits to 1k req/day per IP ‚ÄĒ we mitigate via 24 h `Rails.cache.fetch("geo:#{ip}")`.

**Workaround:** the `GeolocationService` interface is `lookup(ip) ‚Üí Result(country:, city:)`. Swap the body for a MaxMind GeoLite2 implementation ‚ÄĒ local MMDB file (~70 MB), in-process lookups, microsecond latency, no rate limit, no external dependency. One service replacement, zero caller changes.

### 2.5 Click loss vs duplicate clicks

`RecordClickJob` retries transient errors up to 3 times. Retries reuse the original `occurred_at` (captured by the controller *before* enqueue, not generated by the job), so a duplicated row is a true duplicate ‚ÄĒ visible as a count anomaly, not silent corruption. We accept this over the alternative of blocking the redirect on a synchronous log write. After 3 failed attempts the click is dropped and logged via `Rails.logger.warn`.

### 2.6 Rate-limit asymmetry

| Path                 | Limit          | Why |
|----------------------|----------------|-----|
| `POST /short_links`  | 20 req/min/IP  | Form abuse (mass-shortening for spam/phishing) is the realistic threat vector. Tight throttle. |
| `GET /:slug`         | 1000 req/min/IP | A viral link is a *good* problem ‚ÄĒ capping legitimate redirects at creator-rate would be self-defeating. Loose throttle. |

**Workaround if the redirect throttle ever does matter:** a per-slug throttle keyed on the link, not the source IP. That suppresses a single hot link without affecting others.

### 2.7 In-process worker pool sharing

With `SOLID_QUEUE_IN_PUMA=true`, Puma worker threads and Solid Queue's polling fibers share the same Active Record pool. We compensate via `pool: RAILS_MAX_THREADS + 5` (effective default 15). If web latency starts competing with job throughput, the next step is extracting `bin/jobs` as a separate Render service.

## 3. Scalability considerations

At current scale, **write throughput on `clicks` is the bottleneck**; lookup latency on `short_links` is constant (PK + unique-index lookup). Documented growth path, in order:

1. **Cache the slug ‚Üí target_url lookup in Solid Cache** with a 5-minute TTL. Most redirects skip the DB entirely.
2. **Extract `bin/jobs` as its own Render service** so job latency stops competing with web threads.
3. **Move click writes to an append-only sink** (dedicated table + hourly rollup, or a streaming target like Kinesis). Stats reads then hit a materialized view.
4. **Add read replicas for stats pages** once `clicks` exceeds tens of millions of rows.

Concurrent shorten-throughput is bounded by Postgres write throughput on `short_links` and the nanoid retry loop. Sharding or moving to a slug-allocator service is the post-billion-link move.

## 4. Security considerations

| Threat                                              | Mitigation                                                                                                                       |
|-----------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------|
| Malicious `javascript:` / `data:` / `file:` URLs    | Scheme allowlist (`http`/`https`) + length cap (2048) + `URI.parse` rescue, all in the model ‚ÄĒ see `ShortLink#target_url_must_be_http` |
| SSRF via the title fetcher                          | Pre-resolve hostname; reject if any A/AAAA address is in a private/loopback/link-local/multicast/reserved range. Re-run on every redirect hop (‚Č§3) so a 302 to internal infra can't slip through. |
| Form abuse / mass-shortening                        | rack-attack: 20 req/min/IP on `POST /short_links`                                                                                |
| Redirect path abuse                                 | rack-attack: 1000 req/min/IP on `GET /:slug` (loose by design ‚ÄĒ see ¬ß2.6)                                                        |
| CSRF on the form                                    | Rails default CSRF token                                                                                                          |
| XSS via stored `target_url`                         | Rails ERB auto-escapes everything; target URLs are rendered as text via `ShortLinkPresenter#display_target`, never as raw HTML  |
| Clickjacking                                        | `X-Frame-Options: DENY` + CSP `frame-ancestors 'none'`                                                                            |
| MIME sniffing                                       | `X-Content-Type-Options: nosniff`                                                                                                 |
| Mixed-content / downgrade attacks                   | `force_ssl = true` + HSTS (`max-age=1.year; includeSubDomains`)                                                                   |
| Inline-script injection                             | CSP `script-src 'self' https: 'nonce-‚Ä¶'` with a fresh per-request `SecureRandom.base64(16)` nonce                                |
| Excessive title-fetch responses (memory exhaustion) | `TitleFetcher` reads body via streaming `read_body` with a 1 MiB hard cap (truncates in-flight chunks too ‚ÄĒ not just between)    |

What we deliberately don't protect, and why:

- **Public stats pages.** By design for a demo app. Production would gate them.
- **Open-redirect classification.** Brakeman flags `redirect_to short_link.target_url` as an unprotected redirect; this is *the entire purpose* of the service. The actual defense is the scheme allowlist + SSRF guard. The `Redirect` check is disabled in `config/ci.rb` with an explanatory comment.

## 5. PII and retention

Per visit, we store:

- `country` ‚ÄĒ ISO 3166-1 alpha-2 (2 chars; nullable)
- `city` ‚ÄĒ city name (nullable, ‚Č§ 128 chars)
- `ip_hash` ‚ÄĒ `SHA256(ip + ENV['CLICK_IP_SALT'])` as hex (64 chars; not nullable)
- `occurred_at` ‚ÄĒ timestamptz, captured by the controller before enqueue

We deliberately don't store:

- Raw IP address
- Referrer
- User agent

The salt is per-environment (set via Render env var), so hashes from staging don't match hashes from production. Within a single deployment a visitor's hash is stable ‚ÄĒ that's what lets `COUNT(DISTINCT ip_hash)` give us unique-visitor counts (deferred to backlog, but the table is shaped for it).

**Retention story (not implemented for the demo):** a scheduled `PurgeOldClicksJob` would delete clicks older than 90 days. Listed in [`docs/BACKLOG.md`](./BACKLOG.md). The composite `(short_link_id, occurred_at DESC)` index supports a fast purge.
